# Bebochka Telegram Bot

Telegram бот для управления заказами в системе Bebochka. Бот общается с API через HTTP запросы и не имеет прямого доступа к базе данных.

## Архитектура

Бот полностью отделен от backend API и общается с ним только через HTTP REST API:
- ✅ Нет прямых зависимостей от моделей API
- ✅ Нет доступа к базе данных
- ✅ Все операции через HTTP клиент
- ✅ Независимое развертывание

## Возможности

### Для пользователей:
- 📦 Просмотр каталога товаров
- 🛒 Добавление товаров в корзину
- ✅ Оформление заказа
- 📋 Просмотр статуса заказа (В сборке, Ожидает оплату, В пути, Доставлен)
- ❌ Отмена заказа
- 📊 История покупок

### Для администраторов:
- 👨‍💼 Панель администратора
- 📋 Просмотр всех заказов
- 📊 Изменение статусов заказов
- 📈 Статистика продаж

## Установка и настройка

### 1. Получение токена бота

1. Откройте [@BotFather](https://t.me/BotFather) в Telegram
2. Отправьте команду `/newbot`
3. Следуйте инструкциям для создания бота
4. Скопируйте полученный токен

### 2. Настройка конфигурации

Отредактируйте `appsettings.json`:

```json
{
  "Api": {
    "BaseUrl": "http://89.104.67.36:55501/api"
  },
  "TelegramBot": {
    "Token": "YOUR_TELEGRAM_BOT_TOKEN_HERE"
  }
}
```

### 3. Запуск бота

```bash
cd Bebochka.TelegramBot
dotnet restore
dotnet run
```

## Команды бота

### Для всех пользователей:
- `/start` - Начать работу с ботом
- `/catalog` или `/каталог` - Просмотр каталога товаров
- `/cart` или `/корзина` - Просмотр корзины
- `/orders` или `/заказы` - Просмотр заказов

### Для администраторов:
- `/admin` - Панель администратора

## Статусы заказов

- **В сборке** - Заказ создан, готовится к отправке
- **Ожидает оплату** - Заказ ожидает оплаты
- **В пути** - Заказ отправлен, находится в доставке
- **Доставлен** - Заказ доставлен клиенту
- **Отменен** - Заказ отменен

## API Endpoints

Бот использует следующие эндпоинты API:

### Products
- `GET /api/products?sessionId={sessionId}` - Получить список товаров
- `GET /api/products/{id}?sessionId={sessionId}` - Получить товар по ID

### Cart
- `GET /api/cart?sessionId={sessionId}` - Получить корзину
- `POST /api/cart` - Добавить товар в корзину
- `PUT /api/cart/{id}` - Обновить количество
- `DELETE /api/cart/{id}` - Удалить товар из корзины
- `DELETE /api/cart?sessionId={sessionId}` - Очистить корзину

### Orders
- `POST /api/orders` - Создать заказ
- `GET /api/orders/user?userId={userId}` - Получить заказы пользователя
- `GET /api/orders/{id}/public` - Получить заказ по ID
- `POST /api/orders/{id}/cancel` - Отменить заказ
- `GET /api/orders/all` - Получить все заказы (для админа)
- `PUT /api/orders/{id}/status/public` - Обновить статус заказа
- `GET /api/orders/statistics` - Получить статистику

## Примечания

1. Товары с нулевым количеством не отображаются в каталоге
2. Резервирование товаров в корзине действует 20 минут (обрабатывается на стороне API)
3. При отмене заказа товары возвращаются на склад
4. Статусы заказов могут изменять только администраторы
5. Бот использует `chatId` как `sessionId` для корзины
6. Для оформления заказа используется `userId` из Telegram (должен совпадать с User ID в БД)

## Решение проблем

### Бот не отвечает
- Проверьте токен бота в `appsettings.json`
- Убедитесь, что бот запущен
- Проверьте логи на наличие ошибок
- Убедитесь, что API доступен по указанному адресу

### Ошибка подключения к API
- Проверьте `Api:BaseUrl` в `appsettings.json`
- Убедитесь, что API сервер запущен
- Проверьте доступность API по указанному адресу

### Заказы не отображаются
- Убедитесь, что Telegram User ID совпадает с User ID в БД
- Проверьте логи API на наличие ошибок
