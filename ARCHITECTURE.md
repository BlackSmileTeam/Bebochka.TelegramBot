# Архитектура Bebochka.TelegramBot

## Принцип разделения

Telegram бот полностью отделен от backend API и общается с ним **только через HTTP REST API**.

### Преимущества такой архитектуры:

1. **Независимость проектов**
   - Бот может быть развернут отдельно от API
   - Изменения в API не требуют перекомпиляции бота
   - Можно использовать разные технологии и версии .NET

2. **Масштабируемость**
   - Можно запустить несколько экземпляров бота
   - API может обслуживать множество клиентов (веб, мобильные приложения, боты)

3. **Безопасность**
   - Бот не имеет прямого доступа к базе данных
   - Все операции проходят через API с валидацией
   - Можно добавить аутентификацию для API

4. **Простота разработки**
   - Четкое разделение ответственности
   - Легче тестировать и отлаживать
   - Можно использовать разные команды для разработки

## Используемые API Endpoints

### Products
- `GET /api/products?sessionId={sessionId}` - Получить список товаров
- `GET /api/products/{id}?sessionId={sessionId}` - Получить товар по ID

### Cart
- `GET /api/cart?sessionId={sessionId}` - Получить корзину
- `POST /api/cart` - Добавить товар в корзину
- `PUT /api/cart/{id}` - Обновить количество товара
- `DELETE /api/cart/{id}` - Удалить товар из корзины
- `DELETE /api/cart?sessionId={sessionId}` - Очистить корзину

### Orders
- `POST /api/orders` - Создать заказ
- `GET /api/orders/user?userId={userId}` - Получить заказы пользователя
- `GET /api/orders/{id}/public` - Получить заказ по ID (публичный доступ)
- `POST /api/orders/{id}/cancel` - Отменить заказ
- `GET /api/orders/all` - Получить все заказы (для админа)
- `PUT /api/orders/{id}/status/public` - Обновить статус заказа (публичный доступ)
- `GET /api/orders/statistics` - Получить статистику

## Модели данных

Бот использует собственные DTO модели, которые соответствуют API:
- `ProductDto` - информация о товаре
- `CartItemDto` - элемент корзины
- `OrderDto` - информация о заказе
- `OrderItemDto` - элемент заказа
- `CreateOrderDto` - данные для создания заказа
- `CreateOrderItemDto` - элемент для создания заказа

## Конфигурация

Все настройки в `appsettings.json`:
- `Api:BaseUrl` - базовый URL API
- `TelegramBot:Token` - токен Telegram бота

## Зависимости

Бот использует только:
- `Telegram.Bot` - для работы с Telegram API
- `Microsoft.Extensions.Http` - для HTTP клиента
- `Newtonsoft.Json` - для сериализации JSON

**НЕТ зависимостей от:**
- Entity Framework
- Прямого доступа к БД
- Моделей из Bebochka.Api
